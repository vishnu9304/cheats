    1. How to Make a process to run on one specific CPU?

    dd if=/dev/zero of=/dev/null &

    To see how above command is behaving type:

    top -> f [To Add a column] -> j [Last used cpu] -> Come out of the fields section and type (W), so that top will show you the same layout when you run it next time.

    Using taskset, we can make sure that a process will run on one specific CPU.
    Using taskset is not the most efficient way of doing it, Cgroups would be more efficient. But also it is more complicated.

    taskset -pc [-p: Process, -c: cpu-list]

    Example: taskset -pc 3 $(pidof dd)

    2. How to disable a CPU?

    Under /sys we have interfaces that allows us to manipulate the hardware. 

    cd  /sys/bus/cpu/devices  
OR
cd /sys/devices/system/cpu

    The above directory will contain file for every single CPU.

    cd /sys/devices/system/cpu/
ls -l
drwxr-xr-x 8 root root    0 Jan  8 17:35 cpu0
drwxr-xr-x 8 root root    0 Jan  8 17:35 cpu1
drwxr-xr-x 8 root root    0 Jan  8 17:35 cpu2
drwxr-xr-x 8 root root    0 Jan  8 17:35 cpu3

cat cpu1/online

echo "0" > cpu1/online // To bring it offline

    3. How will you kill a Zombie?

    1. Processes can be in different states and depending on the state of the process is in you can manage it or not.
    2. A specific kind of state for processes is zombie.
    3. A zombie is a leftover bit of a dead process that hasn't been cleaned up properly.
    4. A zombie is a process that has completed execution through the exit system call, but it's still in the process table.
    5. zombies do not use any system resources.
    6. The do use a PID though, and the maximum number of PIDs is limited.
    7. So if you have too many zombies you risk running out of PIDs.

    Various Ways to get rid of zombies

    1. kill -s SIGCHLD <ppid>
2. Kill the parent process [Not recommended]
3. If you do manage to kill the parent process, the zombie will become a child of init, and init does occasionally issue the wait system call. 
4. So once the zombie has become a child of init, chances that it's getting reaped are increasing. 
5. The ultimate solution to get rid of all your zombies is to reboot your server.
