    arrays and slices

    Arrays

    An array is a collection of elements that belong to the same type.
    Mixing values of different types, for example an array that contains both strings and integers is not allowed in Go.

    Declaration:

    An array belongs to type [n]T
    n - Number of elements in an array.
    T - Type of each element.

    There are different ways to declare arrays.
    1. var a [5]int32 - int32 array with length 5.
    All elements in an array are automatically assigned the zero value of the array type.
    2. var a = [5]int32{1,2,3,4} or var a = [...]int{1, 2, 3, 4}
    3. a := [5]int{1,2,3,4,5}
    4. a := [...]int{1,2,3}. ... makes the compiler determine the length

    The size of the array is a part of the type.
    [5]int is not equal to [3]int
    [5]int is equal to [5]int

    This is valid:

    package main

import "fmt"

var a [5]int = [5]int{1,2,3,4,5}

func main() {
    var b [5]int
    b = a
    fmt.Println(b)
}

    This is invalid: cannot use a (type [5]int) as type [3]int in assignment.

    package main

import "fmt"

var a [5]int = [5]int{1,2,3,4,5}

func main() {
    var b [3]int
    b = a               // cannot use a (type [5]int) as type [3]int in assignment
    fmt.Println(b)
}

    Arrays are value types

    This means that when they are assigned to a new variable, a copy of the original array is assigned to the new variable.
    If changes are made to the new variable, it will not be reflected in the original array.
    When arrays are passed to functions as parameters, they are passed by value and the original array in unchanged.

    Length of an array: The length of the array is found by passing the array as parameter to the len function len(array).

    Iterating over array

    Using for loop:

    var a [5]int = [5]int {1,2,3,4,5}
for i:=0; i<len(a); i++ {
    fmt.Println(a[i])
}

    Using Range:

    var a [5]int = [5]int {1,2,3,4,5}
    for i, v := range a {
        fmt.Printf("Value at index %d is %d\n", i, v)
}

    Slices

    Although arrays seem to be flexible enough, they come with the restriction that they are of fixed length.
    It is not possible to increase the length of an array.
    This is were slices come into picture.

    Key Points
    Slices are flexible and powerful wrapper on top of an array.
    Slices do not own any data on their own.
    They are the just references to existing arrays.

    Declaration:

    A slice with elements of type T is represented by []T

    Creating a slice 

    a := [5]int{76, 77, 78, 79, 80}
var b := a[1:4]          // a[start:end], creates a slice from array a starting from index [start] to index [end-1]

    Another Way

    c := []int{6, 7, 8}

    Modifying a slice

    A slice does not own any data of its own.
    It is just a representation of the underlying array.
    Any modifications done to the slice will be reflected in the underlying array. 

    package main

import (  
    "fmt"
)

func main() {
    array := [...]int32{1,2,3,4,5,6,7,8}
    slice := array[2:5]
    fmt.Println(array)
    fmt.Println(slice)
    fmt.Println("After incrementing slice")
    for i, _ := range slice {
        slice[i]++
    }
    fmt.Println(array)
    fmt.Println(slice)
}

    Output:

    [1 2 3 4 5 6 7 8]
[3 4 5]
After incrementing slice
[1 2 4 5 6 6 7 8]
[4 5 6]

    length and capacity of a slice

    The length of the slice is the number of elements in the slice.
    The capacity of the slice is the number of elements in the underlying array starting from the index from which the slice is created. 

    fruitarray := [...]string{"apple", "orange", "grape", "mango", "water melon", "pine apple", "chikoo"}

fruitslice := fruitarray[1:3]

fmt.Printf("length of slice %d capacity %d", len(fruitslice), cap(fruitslice))

//length of fruitslice is 2 and capacity is 6

    creating a slice using make

    func make([]T, len, cap) []T can be used to create a slice by passing the type, length and capacity. 

    slice := make([]int,7,7)

    Appending to a slice

    Slices are dynamic and new elements can be appended to the slice using append function.
    The definition of append function is func append(s []T, x ...T) []T 

    cars := []string{"Ferrari", "Honda", "Ford"}
cars = append(cars, "Toyota")

    It is also possible to append one slice to another using the ... operator.

    veggies := []string{"potatoes","tomatoes","brinjal"}
fruits := []string{"oranges","apples"}
food := append(veggies, fruits...)

    Memory Optimisation

    Slices hold a reference to the underlying array.
    As long as the slice is in memory, the array cannot be garbage collected.
    This might be of concern when it comes to memory management.
    Lets assume that we have a very large array and we are interested in processing only a small part of it.
    Henceforth we create a slice from that array and start processing the slice.
    The important thing to be noted here is that the array will still be in memory since the slice references it.

    One way to solve this problem is to use the copy function.

    func copy(dst, src []T) int

    countries := []string{"USA", "Singapore", "Germany", "India", "Australia"}
neededCountries := countries[:len(countries)-2]
countriesCpy := make([]string, len(neededCountries))
copy(countriesCpy, neededCountries)
