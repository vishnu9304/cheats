    structs

    1. A structure is a user defined type which represents a collection of fields.
    2. It can be used in places where it makes sense to group the data into a single unit rather than maintaining each of them as separate types.

    Example: Named Structure Type

    type Employee struct {
    FirstName string
    LastName string
    Age int
}

    Above struct can also be written as.

    type Employee struct {
    FistName LastName string
    Age int
}

    Example: Anonymous Structure without declaring a new type

    var employee struct {
    FistName, LastName string
}

(or)

employee := struct {
    FirstName, LastName string
}{
    FirstName: "vishnu",
    LastName: "pradeep",
}

    Zero value of a structure: When a struct is defined and it is not explicitly initialised with any value, the fields of the struct are assigned their zero values by default.

    Pointers to a struct

    type Employee struct {
    FirstName, LastName string
}

func main() {
    e1 := &Employee{
        FirstName: "vishnu",
        LastName: "pradeep",
    }
    fmt.Println(e1.FirstName)
    fmt.Println(*e1.FirstName)
}

    The language gives us the option to use e1.FirstName instead of the explicit dereference (*emp8).firstName to access the firstName field.

    Anonymous fields: It is possible to create structs with fields which contain only a type without the field name. 

    type Person struct {  
    string
    int
}

    Even though an anonymous fields does not have a name, by default the name of a anonymous field is the name of its type.

    Promoted fields: 

    package main

import "fmt"

type Employee struct {
    Name    string
    Email   string
    Address Address
}

type Address struct {
    Location string
}

func main() {
    e1 := Employee{
        Name:  "vishnu",
        Email: "vi@yahoo.com",
        Address: Address{
            Location: "Blr",
        },
    }
    fmt.Println(e1.Location)       // e1.Location undefined (type Employee has no field or method Location)
    fmt.Println(e1.Address.Location)
}

    Fields that belong to a anonymous struct field in a structure are called promoted fields since they can be accessed as if they belong to the structure which holds the anonymous struct field.
