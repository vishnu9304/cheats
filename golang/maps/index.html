    maps

    What is a map?

    A map is a builtin type in Go which associates a value to a key.
    The value can be retrieved using the corresponding key. 

    map[TYPE OF KEY] TYPE OF VALUE

    How to create a map?

    make(map[TYPE OF KEY] TYPE OF VALUE)

personSalary := make(map[string]int)
    1. The zero value of a map is nil.
    2. If you try to add items to nil map, a run time panic will occur.
    3. Hence the map has to be initialized using make function.

    Example:

    package main

import (
    "fmt"
)

func main() {
    var personSalary map[string]int // declared not initalized
    if personSalary == nil {
        fmt.Println("map is nil. Going to make one.")
        personSalary = make(map[string]int) // initalized
    }
}

    Adding items to a map

    personSalary := make(map[string]int)
personSalary["steve"] = 12000
personSalary["jamie"] = 15000
personSalary["mike"] = 9000

    It is also possible to initialize a map during declaration itself.

    personSalary := map[string]int {
    "steve": 12000,
    "jamie": 15000,
}
personSalary["mike"] = 9000

    Accessing the the items in map

    map["KEY"]
personSalary["steve"]

    What will happen if a element is not present?
    The map will return the zero value of the type of that element.

    What if we want to know whether a key is present in a map or not? 

    value, ok := map[key]

    Deleting items

    delete(map, key)

    Length of the map

    len(MAP)

    Maps are reference types

    1. Similar to slices, maps are reference types.
    2. When a map is assigned to a new variable, they both point to the same internal data structure.
    3. Similar is the case when maps are passed as parameters to functions. When any change is made to the map inside the function, it will be visible to the caller.

    Maps equality

    1. Maps can't be compared using the == operator.
    2. The == can be only used to check if a map is nil.
